Sprint Review Template

Sprint Review Meeting 邀请模板

在 sprint review meeting 上总结回顾一下这个 sprint 所做的 features 和 user stories

请大家预先检查一下看板上未完成的 user story 以及 bug ，及时更新它们的状态

如果有可以演示的 user story，请相关同事准备一下，给团队做个快速的演示。

如果有完成的 feature, 也请 feature owner 准备一下 feature 的完成总结

Sprint Review Meeting 议程模板

会议目标

  • 展示 Sprint 的成果
  • 收集利益相关者的反馈
  • 讨论和评估 Sprint 的成功和不足
  • 为下一个 Sprint 确定改进方向

会议时间

  • 日期: [日期]
  • 时间: [开始时间] - [结束时间]
  • 时长: [时长] 分钟（通常不超过 2 小时，对于 2 周的 Sprint）

会议地点

  • [会议室/线上会议链接]

参会人员

  • Scrum Master: [姓名]
  • 产品负责人 (Product Owner): [姓名]
  • 开发团队成员: [姓名1], [姓名2], [姓名3], ...
  • 利益相关者: [姓名1], [姓名2], [姓名3], ...

会议议程

1. 会议开场 (5 分钟)

  • 主持人: [Scrum Master 姓名]
  • 内容:
  • 欢迎所有参会人员
  • 简要介绍会议目的和预期成果
  • 确认会议议程和时间安排

2. 回顾 Sprint 目标 (10 分钟)

  • 主持人: [产品负责人 姓名]
  • 内容:
  • 产品负责人回顾 Sprint 的目标
  • 展示 Sprint 的主要成果和完成的工作
  • 确认 Sprint 的主要交付物

3. 展示 Sprint 成果 (30 分钟)

  • 主持人: [开发团队成员]
  • 内容:
  • 团队成员轮流展示 Sprint 中完成的用户故事和功能
  • 演示新功能或改进的系统
  • 产品负责人可以补充说明每个交付物的价值和影响

4. 收集反馈 (20 分钟)

  • 主持人: [Scrum Master 姓名]
  • 内容:
  • 邀请利益相关者提供反馈
  • 记录所有反馈，包括正面和负面的
  • 确认反馈的理解和记录

5. 讨论 Sprint 成功和不足 (20 分钟)

  • 主持人: [Scrum Master 姓名]
  • 内容:
  • 团队讨论 Sprint 的成功之处
  • 分析 Sprint 中遇到的问题和挑战
  • 识别改进的机会

6. 为下一个 Sprint 确定改进方向 (15 分钟)

  • 主持人: [产品负责人 姓名]
  • 内容:
  • 基于收集到的反馈和团队的讨论，确定下一个 Sprint 的改进方向
  • 讨论如何调整 Product Backlog 以反映新的需求和优先级
  • 确认下一步的行动计划

7. 会议总结 (10 分钟)

  • 主持人: [Scrum Master 姓名]
  • 内容:
  • 确认 Sprint 成果和反馈
  • 感谢所有参会人员的参与和贡献
  • 确认下次会议的时间和地点
